# purl2repo v2.0.2

`purl2repo` 2.0.2 improves bulk-resolution controls and makes repository
validation status a first-class result field.

## Highlights

- Added resolver settings and CLI flags to disable repository URL validation,
  deps.dev fallback, and scraper fallback for faster first-party or bulk runs.
- Added `repository_validated` and `repository_validation_status` to
  `ResolutionResult` and JSON output.
- Added optional bounded parallelism with
  `Resolver.resolve_many(..., max_workers=N)`.
- Tightened Maven parent POM SCM scoring so inherited build metadata is not
  treated like artifact-owned SCM metadata.
- Tightened fallback scraping so weak HTML-derived links remain low confidence.
- Capped inconclusive repository URL validation below medium confidence.

## Compatibility

This release extends the v2 dataclass and JSON output contract. Existing code
that reads `repository_url`, `canonical_repository`, `confidence`, evidence, and
warnings continues to work. Code that constructs `ResolutionResult` directly can
omit the new validation fields; they default to `False` and `unknown`.
